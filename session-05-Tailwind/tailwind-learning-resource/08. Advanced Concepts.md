## Introduction to Tailwind CSS v4

Tailwind CSS v4 represents a major evolution from previous versions. Key changes:

- **CSS-first configuration**: Everything in CSS files, no JavaScript config
- **Single import**: Just `@import "tailwindcss";`
- **Automatic content detection**: No need to specify content paths
- **Enhanced performance**: Only includes used styles
- **Modern CSS features**: Uses CSS custom properties extensively

## @theme - Design System Variables

The `@theme` directive defines your project's design tokens that become available as utility classes.

```css
@theme {
  /* Colors */
  --color-primary: #3b82f6;
  --color-secondary: #10b981;
  --color-accent: #f59e0b;

  /* Spacing */
  --spacing-xs: 0.25rem;
  --spacing-sm: 0.5rem;
  --spacing-md: 1rem;

  /* Typography */
  --font-family-sans: "Inter", sans-serif;
  --font-family-display: "Outfit", sans-serif;
  --font-size-base: 1rem;
  --line-height-base: 1.5;

  /* Shadows */
  --shadow-sm: 0 1px 2px 0 rgb(0 0 0 / 0.05);
  --shadow-lg: 0 10px 15px -3px rgb(0 0 0 / 0.1);

  /* Border radius */
  --radius-sm: 0.25rem;
  --radius-md: 0.5rem;
  --radius-lg: 1rem;

  /* Animations */
  --animate-bounce: bounce 1s infinite;
  --animate-spin: spin 1s linear infinite;

  @keyframes bounce {
    0%,
    100% {
      transform: translateY(0);
    }
    50% {
      transform: translateY(-10px);
    }
  }

  @keyframes spin {
    from {
      transform: rotate(0deg);
    }
    to {
      transform: rotate(360deg);
    }
  }
}
```

### Available as Utility Classes

Theme variables automatically become Tailwind utilities:

```html
<!-- Colors -->
<div class="bg-primary text-secondary border-accent"></div>

<!-- Spacing -->
<div class="p-xs m-sm gap-md"></div>

<!-- Typography -->
<div class="font-sans font-display text-base leading-base"></div>

<!-- Shadows and radius -->
<div class="shadow-sm shadow-lg rounded-sm rounded-lg"></div>

<!-- Animations -->
<div class="animate-bounce animate-spin"></div>
```

## @layer Directives

Tailwind CSS organizes styles into layers with specific specificity order:

### @layer base - Global Styles

Applies to HTML elements globally. Runs first (lowest specificity).

```css
@layer base {
  * {
    @apply border-border;
  }

  body {
    @apply bg-background text-foreground font-sans;
  }

  h1,
  h2,
  h3,
  h4,
  h5,
  h6 {
    @apply font-display font-bold tracking-tight;
  }

  a {
    @apply text-primary hover:text-primary/80 transition-colors;
  }

  button {
    @apply cursor-pointer;
  }
}
```

### @layer components - Reusable Components

Component classes that combine multiple utilities. Medium specificity.

```css
@layer components {
  .btn {
    @apply inline-flex items-center justify-center rounded-md px-4 py-2 text-sm font-medium transition-colors focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2 disabled:opacity-50 disabled:pointer-events-none;
  }

  .btn-primary {
    @apply btn bg-primary text-primary-foreground hover:bg-primary/90;
  }

  .btn-secondary {
    @apply btn bg-secondary text-secondary-foreground hover:bg-secondary/80;
  }

  .card {
    @apply rounded-lg border bg-card text-card-foreground shadow-sm;
  }

  .input {
    @apply flex h-10 w-full rounded-md border border-input bg-background px-3 py-2 text-sm ring-offset-background placeholder:text-muted-foreground focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2 disabled:cursor-not-allowed disabled:opacity-50;
  }
}
```

### @layer utilities - Custom Utilities

High-specificity utilities that override everything else.

```css
@layer utilities {
  .text-balance {
    text-wrap: balance;
  }

  .bg-gradient-radial {
    background-image: radial-gradient(var(--tw-gradient-stops));
  }

  .animate-fade-in {
    animation: fadeIn 0.5s ease-in-out;
  }

  @keyframes fadeIn {
    from {
      opacity: 0;
    }
    to {
      opacity: 1;
    }
  }
}
```

## @apply - Composing Utilities

`@apply` combines multiple utility classes into reusable component classes.

```css
/* Basic component */
.btn {
  @apply px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600;
}

/* Complex component with responsive variants */
.card {
  @apply p-6 bg-white rounded-lg shadow-md border border-gray-200;
  @apply md:p-8 lg:shadow-lg;
}

/* Component with state variants */
.alert {
  @apply p-4 rounded-md;
}

.alert-info {
  @apply alert bg-blue-50 text-blue-800 border-blue-200;
}

.alert-success {
  @apply alert bg-green-50 text-green-800 border-green-200;
}

/* Using @apply with @layer */
@layer components {
  .hero-section {
    @apply min-h-screen flex items-center justify-center bg-gradient-to-br from-blue-500 to-purple-600;
    @apply text-white text-center px-4;
  }

  .hero-title {
    @apply text-4xl md:text-6xl font-bold mb-4;
    @apply bg-clip-text text-transparent bg-gradient-to-r from-white to-gray-300;
  }
}
```

## Dark Mode Implementation

Tailwind CSS v4 supports dark mode through the `dark:` prefix.

### CSS Setup

```css
@theme {
  --color-background: #ffffff;
  --color-foreground: #000000;
  --color-muted: #f8f9fa;
  --color-muted-foreground: #6c757d;

  /* Dark mode overrides */
  .dark {
    --color-background: #1a1a1a;
    --color-foreground: #ffffff;
    --color-muted: #2d2d2d;
    --color-muted-foreground: #adb5bd;
  }
}

@layer base {
  html {
    @apply bg-background text-foreground;
  }
}

@layer components {
  .card {
    @apply bg-background border border-muted text-foreground;
    @apply dark:bg-muted dark:border-muted-foreground/20;
  }
}
```

### HTML Implementation

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <title>Dark Mode Example</title>
    <link rel="stylesheet" href="output.css" />
  </head>
  <body class="bg-background text-foreground">
    <!-- Dark mode toggle -->
    <button
      onclick="document.documentElement.classList.toggle('dark')"
      class="p-2 bg-muted rounded"
    >
      🌙 Toggle Dark Mode
    </button>

    <!-- Content that respects dark mode -->
    <div class="card p-6 rounded-lg">
      <h2 class="text-xl font-bold">Card Title</h2>
      <p class="text-muted-foreground">This card changes with dark mode.</p>
    </div>
  </body>
</html>
```

### JavaScript Enhancement

```javascript
// Check for saved theme preference or default to light mode
const theme = localStorage.getItem("theme") || "light";

// Apply the theme
if (theme === "dark") {
  document.documentElement.classList.add("dark");
}

// Toggle function
function toggleDarkMode() {
  const html = document.documentElement;
  const isDark = html.classList.toggle("dark");
  localStorage.setItem("theme", isDark ? "dark" : "light");
}
```

## Responsive Design

Use responsive prefixes with any utility class.

```css
@layer components {
  .responsive-grid {
    @apply grid grid-cols-1 gap-4;
    @apply sm:grid-cols-2 md:grid-cols-3 lg:grid-cols-4;
  }

  .responsive-text {
    @apply text-sm sm:text-base md:text-lg lg:text-xl;
  }

  .responsive-spacing {
    @apply p-4 sm:p-6 md:p-8;
  }
}
```

### Breakpoint Reference

- `sm:` - 640px and up
- `md:` - 768px and up
- `lg:` - 1024px and up
- `xl:` - 1280px and up
- `2xl:` - 1536px and up

## Custom Animations

Define custom animations in `@theme` and use them as utilities.

```css
@theme {
  --animate-fade-in: fadeIn 0.5s ease-out;
  --animate-slide-up: slideUp 0.6s ease-out;
  --animate-scale-in: scaleIn 0.4s ease-out;

  @keyframes fadeIn {
    from {
      opacity: 0;
    }
    to {
      opacity: 1;
    }
  }

  @keyframes slideUp {
    from {
      opacity: 0;
      transform: translateY(20px);
    }
    to {
      opacity: 1;
      transform: translateY(0);
    }
  }

  @keyframes scaleIn {
    from {
      opacity: 0;
      transform: scale(0.9);
    }
    to {
      opacity: 1;
      transform: scale(1);
    }
  }
}

@layer components {
  .fade-in {
    @apply animate-fade-in;
  }

  .slide-up {
    @apply animate-slide-up;
  }

  .scale-in {
    @apply animate-scale-in;
  }
}
```

## CSS-in-JS Approach

Tailwind v4's CSS-first approach offers advantages:

### Advantages

1. **Build-time optimization**: Only used styles are included
2. **Better performance**: No runtime CSS generation
3. **Static analysis**: Easier to optimize and tree-shake
4. **Familiar CSS**: No learning curve for CSS developers

### Migration from v3

```javascript
// tailwind.config.js (v3)
module.exports = {
  theme: {
    extend: {
      colors: {
        brand: "#3b82f6",
      },
      fontFamily: {
        display: ["Outfit", "sans-serif"],
      },
    },
  },
};
```

```css
/* input.css (v4) */
@import "tailwindcss";

@theme {
  --color-brand: #3b82f6;
  --font-display: "Outfit", sans-serif;
}
```

## Build Process

### One-time Build

```bash
npx @tailwindcss/cli -i ./src/input.css -o ./src/output.css
```

### Watch Mode (Development)

```bash
npx @tailwindcss/cli -i ./src/input.css -o ./src/output.css --watch
```

### With npm Scripts

Add to `package.json`:

```json
{
  "scripts": {
    "build": "tailwindcss -i ./src/input.css -o ./src/output.css",
    "dev": "tailwindcss -i ./src/input.css -o ./src/output.css --watch"
  }
}
```

Then run:

```bash
npm run build
npm run dev
```

## Best Practices

### 1. Use Semantic Component Classes

```css
/* Good */
.btn-primary {
  @apply bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600;
}

/* Avoid */
.px-4 {
  @apply py-2 rounded bg-blue-500 text-white hover:bg-blue-600;
}
```

### 2. Leverage @theme for Consistency

```css
@theme {
  --color-primary: #3b82f6;
  --color-primary-hover: #2563eb;
  --spacing-card: 1.5rem;
}

@layer components {
  .card {
    @apply p-card bg-white border border-gray-200 rounded-lg;
  }

  .btn-primary {
    @apply bg-primary hover:bg-primary-hover text-white px-4 py-2 rounded;
  }
}
```

### 3. Use @layer for Organization

- `@layer base`: Global resets and defaults
- `@layer components`: Reusable UI components
- `@layer utilities`: One-off utilities

### 4. Optimize for Performance

- Only include used utilities in production
- Use component classes for repeated patterns
- Leverage Tailwind's purging automatically

### 5. Dark Mode First

```css
/* Define light mode in @theme, dark mode as overrides */
@theme {
  --color-bg: #ffffff;
  --color-text: #000000;

  .dark {
    --color-bg: #1a1a1a;
    --color-text: #ffffff;
  }
}
```

### 6. Responsive Design Patterns

```css
@layer components {
  .container {
    @apply mx-auto px-4;
    @apply sm:px-6 lg:px-8;
    @apply max-w-7xl;
  }

  .grid-responsive {
    @apply grid grid-cols-1 gap-6;
    @apply md:grid-cols-2 lg:grid-cols-3;
  }
}
```

## Troubleshooting

### Styles Not Applying

1. Check if CSS is being rebuilt after changes
2. Verify class names are correct
3. Check browser dev tools for CSS conflicts
4. Ensure `@import "tailwindcss";` is at the top

### Dark Mode Not Working

1. Check if `dark` class is added to `html` element
2. Verify dark mode utilities are defined
3. Test with browser dev tools

### Build Errors

1. Ensure all dependencies are installed
2. Check for syntax errors in CSS
3. Verify file paths in build command
